# 功能设计类需求澄清模板

## 典型场景

- "添加登录功能"
- "做一个管理后台"
- "支持导出报表"
- "添加消息通知"
- "实现搜索功能"

## 标准澄清问题

### 1. 功能范围

- **核心功能是什么？**
  - 必须有的功能？
  - 可选/延后的功能？
  - 明确不做什么？

### 2. 用户场景

- **目标用户是谁？**
  - 内部用户 vs 外部用户？
  - 用户角色/权限？
  - 典型使用流程？

### 3. 业务逻辑

- **核心业务规则是什么？**
  - 数据如何流转？
  - 有哪些状态/状态机？
  - 有哪些约束条件？

### 4. 技术约束

- **技术栈是什么？**
  - 前端框架？
  - 后端框架？
  - 数据库/存储？

### 5. 验收标准

- **如何验证功能符合预期？**
  - 关键用户路径？
  - 边界情况？
  - 异常处理？

## 示例 AskUserQuestion 调用

```python
AskUserQuestion(
    questions=[
        {
            "question": "这个管理后台需要包含哪些核心功能？",
            "header": "功能范围",
            "options": [
                {"label": "数据查看", "description": "列表、详情、筛选"},
                {"label": "数据管理", "description": "增删改操作"},
                {"label": "权限控制", "description": "角色、权限管理"},
                {"label": "数据分析", "description": "报表、统计图表"}
            ],
            "multiSelect": true
        },
        {
            "question": "预期的用户量级是多少？",
            "header": "用户规模",
            "options": [
                {"label": "小规模", "description": "1-10 个内部用户"},
                {"label": "中等规模", "description": "10-100 个用户"},
                {"label": "大规模", "description": "100+ 个用户"}
            ],
            "multiSelect": false
        }
    ]
)
```

## 常见补充问题

### UI/UX
- 有设计稿/原型吗？
- 是否需要响应式设计？
- 是否有品牌/样式要求？

### 数据
- 需要管理哪些数据？
- 数据从哪里来？
- 需要哪些查询/筛选能力？

### 安全
- 是否需要认证/授权？
- 是否有敏感数据？
- 是否需要审计日志？

### 集成
- 是否需要集成现有系统？
- 是否需要调用外部 API？
- 是否需要数据同步？
