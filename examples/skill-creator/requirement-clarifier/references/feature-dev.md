# 新功能开发澄清模板

适用于功能设计、架构决策等需要规划的新功能开发任务。

## 核心澄清问题

### 范围与优先级
1. **功能目标**
   - 这个功能要解决什么问题？
   - 目标用户是谁？
   - 预期带来什么价值？

2. **功能范围**
   - 核心功能是什么？
   - 哪些是 MVP 必须的，哪些可以后续迭代？
   - 是否需要分阶段实现？

3. **优先级**
   - 是否有其他功能依赖于此功能？
   - 是否有上线时间要求？

### 技术实现
1. **架构设计**
   - 需要哪些组件或模块？
   - 组件之间如何交互？
   - 是否需要与其他系统集成？

2. **技术选型**
   - 是否有技术栈限制？
   - 是否需要引入新的依赖或服务？
   - 是否需要考虑扩缩容？

3. **实现方式选项**
   - 是否有多种架构方案可选？
   - 不同方案的权衡是什么（复杂度、性能、维护成本）？

### 质量标准
1. **验收标准**
   - 功能完成的定义是什么？
   - 需要哪些验收测试？
   - 性能、安全、可用性要求是什么？

2. **可观测性**
   - 需要哪些监控指标？
   - 需要哪些日志？
   - 如何衡量功能效果？

3. **边界条件**
   - 功能的上限和下限是什么？
   - 异常情况如何处理？
   - 降级方案是什么？

### 约束条件
1. **资源约束**
   - 开发时间限制？
   - 团队技能限制？
   - 基础设施限制？

2. **业务约束**
   - 是否有合规要求？
   - 是否有成本限制？
   - 是否有数据隐私要求？

## 常见澄清场景

### 场景 1：全新功能开发
用户说："添加一个导出功能"

澄清问题：
- 需要导出什么数据？
- 导出格式要求（CSV、Excel、PDF）？
- 数据量规模（单个文件大小、记录数）？
- 导出方式（同步、异步、后台任务）？
- 是否需要权限控制？

### 场景 2：集成第三方服务
用户说："集成支付功能"

澄清问题：
- 使用哪个支付提供商（Stripe、支付宝、微信支付）？
- 需要支持哪些支付方式？
- 如何处理支付失败？
- 是否需要订阅/定期支付？
- 如何处理退款？
- 安全性要求（PCI DSS）？

### 场景 3：搜索功能
用户说："添加搜索功能"

澄清问题：
- 搜索范围（哪些字段/实体）？
- 搜索类型（精确匹配、模糊搜索、全文检索）？
- 是否需要高级搜索（筛选、排序、分页）？
- 数据量规模（是否需要搜索引擎）？
- 搜索性能要求？
- 是否需要搜索历史/建议？

### 场景 4：实时通知
用户说："添加通知功能"

澄清问题：
- 通知类型（邮件、短信、推送、站内信）？
- 触发条件是什么？
- 通知频率控制（防打扰）？
- 是否需要实时推送（WebSocket）？
- 通知模板管理？
- 用户偏好设置？
