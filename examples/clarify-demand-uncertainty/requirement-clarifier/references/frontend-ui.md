# 前端/UI 任务澄清模板

适用于界面设计、组件开发、样式调整等前端任务。

## 核心澄清问题

### 范围与优先级
1. **具体要实现什么 UI 元素或功能？**
   - 是新建页面/组件，还是修改现有页面/组件？
   - 涉及哪些具体的 UI 元素（按钮、表单、列表、卡片等）？

2. **视觉设计要求是什么？**
   - 是否有设计稿或参考示例？
   - 需要遵循哪些设计规范（颜色、字体、间距等）？
   - 是否需要响应式设计？支持哪些屏幕尺寸？

3. **功能范围边界**
   - 需要哪些交互功能（点击、悬停、拖拽等）？
   - 是否需要数据加载和状态管理？
   - 哪些功能是核心必须的，哪些可以后续迭代？

### 技术实现
1. **技术栈选择**
   - 使用什么前端框架（React/Vue/Angular/原生等）？
   - UI 组件库是否有要求（shadcn/ui、Material-UI、Ant Design 等）？
   - 样式方案偏好（CSS、Tailwind、CSS-in-JS 等）？

2. **实现方式选项**
   - 从零开发 vs 使用现有组件库？
   - 单文件组件 vs 多文件拆分？
   - 是否需要考虑与现有路由/状态管理的集成？

### 质量标准
1. **交互体验**
   - 加载状态如何展示（loading、骨架屏等）？
   - 错误状态如何处理？
   - 空状态如何展示？

2. **边界条件**
   - 超长文本/标题如何处理？
   - 图片加载失败如何处理？
   - 网络请求失败如何处理？

3. **可访问性**
   - 是否需要考虑无障碍访问（ARIA 标签、键盘导航等）？
   - 是否需要支持屏幕阅读器？

### 约束条件
1. **兼容性要求**
   - 需要支持哪些浏览器？
   - 是否需要支持旧版浏览器？

2. **性能要求**
   - 首屏加载时间要求？
   - 动画/交互性能要求？

## 常见澄清场景

### 场景 1：模糊的设计需求
用户说："改进这个页面的样式"

澄清问题：
- 具体想要改进哪些方面（布局、颜色、字体、间距）？
- 是否有参考的设计风格或网站？
- 改进的目标是什么（更美观、更易用、更符合品牌等）？

### 场景 2：组件开发
用户说："创建一个表单组件"

澄清问题：
- 表单包含哪些字段（类型、标签、验证规则）？
- 提交按钮的行为是什么（跳转、弹窗、刷新等）？
- 是否需要实时验证？
- 错误提示如何展示？

### 场景 3：响应式设计
用户说："让这个页面在移动端也能用"

澄清问题：
- 需要支持哪些设备尺寸（手机、平板、桌面）？
- 移动端是否需要不同的布局或交互方式？
- 是否需要横屏/竖屏适配？
