### 核心案例
1. **营销活动数据分析**（主案例）
   - 传统方式：3轮对话，1500字提示词，15分钟
   - Skill方式：1轮对话，80字提示词，3分钟
   - 效率提升：5倍

2. **需求澄清Skill**（实战练习）
   - 问题：AI经常对模糊需求做错误假设

   - 工具：使用skill-creator创建

## 🚀 使用流程

### 步骤1：准备阶段
1. **阅读课程大纲**
   - 理解课程目标和教学理念
   - 熟悉每个部分的知识点
   - 准备所需的演示材料

2. **准备演示环境**
   - 安装Claude Code
   - 准备analyzing-marketing-campaign案例数据
   - 准备skill-creator演示

3. **准备补充材料**
   - 下载FlySkill项目：https://github.com/Agent101/FlySkill
   - 准备marketing_data.csv示例数据
   - 准备演示录屏（作为备用）

### 步骤2：制作PPT
**推荐工具**：Gamma.app / Beautiful.ai / Canva

#### 使用Gamma.app（推荐）
1. 访问 https://gamma.app
2. 选择"Create from Text"
3. 复制粘贴`《Agent Skill 重构传统SOP》PPT讲稿.md`的内容
4. 选择模板风格（推荐：Modern / Tech）
5. AI自动生成PPT（约5-10分钟）
6. 手动调整：
   - 添加代码块的语法高亮
   - 调整对比页的左右布局
   - 添加流程图和示意图
   - 优化表格样式

#### 手动调整重点
- **第3页**（对比页）：使用左右分栏布局
- **第15页**（三层架构）：使用流程图或表格
- **第27页**（文件结构）：使用树形图
- **第35页**（对比总结）：使用表格+数据可视化

### 步骤3：试讲与优化
1. **第一次试讲**（全程录像）
   - 检查时间控制（是否在80分钟内）
   - 检查衔接是否流畅
   - 标注容易讲错的地方

2. **优化内容**
   - 删减冗余内容
   - 补充不清晰的说明
   - 调整PPT动画和布局

3. **第二次试讲**
   - 验证优化效果
   - 准备Q&A问题库
   - 完善讲师备注

### 步骤4：正式授课
1. **课前准备**（30分钟前）
   - 测试设备和网络
   - 打开所有演示环境
   - 播放开场音乐（可选）

2. **授课中**
   - 严格控制时间（建议每部分设闹钟提醒）
   - 关键演示使用录屏备用（避免现场问题）
   - 记录学员提出的新问题

3. **课后跟进**
   - 收集学员反馈
   - 分享课程资料（大纲、示例Skill、参考链接）
   - 建立学习社群

## 💡 教学技巧

### 讲解风格参考
课程设计参考了"会生老师AI的办公提效课"，特点：
- ✅ **节奏清晰**：每个知识点讲透后再进入下一个
- ✅ **步骤明确**：先讲什么，再讲什么，逻辑清晰
- ✅ **方法论拆分**：不只讲操作，更讲背后的方法论
- ✅ **案例丰富**：每个知识点都有具体案例支撑

### 关键讲解点

#### 1. 第3页（对比页）- 全课程最重要
**讲解重点**：
- 这一页是全课程的价值主张
- 要让学员产生"哇，原来可以这样"的感觉
- 强调数字对比：15分钟→3分钟，1500字→80字

**讲解技巧**：
- 先展示传统方式的痛苦（语速可以慢一些，制造"冗长"的感觉）
- 再展示Skill方式的简洁（语速加快，制造"爽快"的感觉）
- 停顿3秒，让对比产生冲击力

#### 2. 第15页（三层架构）- 最核心的技术原理
**讲解重点**：
- 这是Skill的核心设计哲学
- 要让学员理解"为什么要分层"

**讲解技巧**：
- 用比喻：像图书馆的目录（Level 1）→ 书架（Level 2）→ 书的内容（Level 3）
- 强调Token消耗数字：50 → 2000 → 5000
- 解释"按需加载"的价值：就像点菜，不是一次性把厨房的菜都端上来

#### 3. 第30页（好坏description对比）- 实战能力的关键
**讲解重点**：
- description是Skill能否准确触发的关键
- 要让学员建立"好description"的直觉

**讲解技巧**：
- 先展示坏例子，让学员感受"确实不好"
- 再展示好例子，指出具体好在哪里
- 引导学员思考：如果你是AI，看到这个描述能判断吗？

#### 4. 第40-41页（实战演示）- 最有价值的环节
**讲解重点**：
- 让学员跟着一起操作
- 建立"我也能做到"的信心

**讲解技巧**：
- 放慢语速，让学员跟上
- 每一步都停顿，确认学员完成了
- 准备助教协助遇到问题的学员
- 预留5分钟的troubleshooting时间

### 互动设计

#### 提问时机
1. **第11页后**："大家平时工作中，有哪些需要重复输入的长提示词？"
   - 收集3-5个真实场景
   - 后续可以作为案例讲解

2. **第25页后**："谁已经成功安装并看到Skill了？举个手"
   - 确认学员进度
   - 协助遇到问题的学员

3. **第43页后**："测试成功的同学，你的Skill被触发了吗？"
   - 收集测试结果
   - 讨论触发不准确的原因

#### 课堂练习
1. **第22页后**（5分钟）
   - 任务：安装analyzing-marketing-campaign Skill
   - 助教巡场协助

2. **第40-42页**（12分钟）
   - 任务：跟着讲师创建clarify-demand-uncertainty Skill
   - 助教巡场协助

## 🔍 常见问题解答

### Q1: 学员的技术水平不一致怎么办？
**A**：
- 课前发送预习材料（Claude Code安装指南）
- 课中安排助教协助技术问题
- 关键步骤准备录屏，学员可以课后回看
- 课后提供详细文档和视频教程

### Q2: 80分钟时间不够怎么办？
**A**：
- 准备"快进"版本：省略部分细节讲解
- 将实战练习改为演示（不让学员跟着做）
- 课后提供完整视频录播

### Q3: 学员提出课程范围外的问题怎么办？
**A**：
- 记录问题，承诺课后单独解答
- 在课程社群中统一回答
- 整理成FAQ文档

### Q4: 演示环节出现技术问题怎么办？
**A**：
- 使用提前录制的演示视频
- 或跳过演示，直接展示结果
- 课后补充完整演示视频

### Q5: 学员的场景和案例差异很大怎么办？
**A**：
- 使用通用的方法论讲解（6步改造法）
- 课堂上收集2-3个学员的真实场景，简要分析
- 课后提供一对一咨询（可选）
