## 🎯 关键环节准备

### 第3页：价值对比（5分钟）
**为什么重要**：决定学员是否认可课程价值

**准备重点**：
- [ ] 对比数据要背下来（1500字→80字，15分钟→3分钟）
- [ ] 语速控制：传统方式慢讲，Skill方式快讲（制造对比）
- [ ] 准备停顿3秒，让冲击力产生

**可能的问题**：
- Q: "这个数据是真实的吗？"
- A: 准备实际案例截图证明

---

### 第15页：三层架构（7分钟）
**为什么重要**：决定学员是否理解技术原理

**准备重点**：
- [ ] 画出三层架构示意图（建议用白板或动画）
- [ ] 准备比喻：图书馆的目录→书架→书的内容
- [ ] 强调Token数字：50 → 2000 → 5000

**可能的问题**：
- Q: "为什么不一次性加载所有内容？"
- A: 类比点菜，不是把厨房的菜都端上来

---

### 第20-25页：环境搭建（15分钟）
**为什么重要**：决定学员能否跟上实战环节

**准备重点**：
- [ ] 准备助教（至少1人）巡场协助
- [ ] 提前测试安装流程，标注常见问题
- [ ] 准备多个安装方法（Windows/Mac/Linux）
- [ ] 准备"跳过"方案（如学员安装失败）

**可能的问题**：
- Q: "我没有Anthropic API密钥怎么办？"
- A: 提供临时测试账号或使用claude.ai网页版

---

### 第40-43页：实战练习（15分钟）
**为什么重要**：决定学员能否实际创建Skill

**准备重点**：
- [ ] 自己先完整走一遍创建流程
- [ ] 准备skill-creator的每个问题的标准答案
- [ ] 预留5分钟troubleshooting时间
- [ ] 准备"快速通道"（提供预创建的SKILL.md让学员修改）

**可能的问题**：
- Q: "我创建的Skill不触发怎么办？"
- A: 检查description是否包含关键词，现场调整

## 🗣️ 讲解技巧准备

### 语速控制
- [ ] **正常语速**：理论讲解部分（150字/分钟）
- [ ] **放慢语速**：关键概念和步骤说明（100字/分钟）
- [ ] **加快语速**：对比传统方式时（制造"冗长"感）

### 停顿技巧
- [ ] **价值对比后**：停顿3秒，让冲击力产生
- [ ] **提出问题后**：停顿5秒，等待学员思考
- [ ] **演示前**：停顿2秒，确认学员准备好

### 互动设计
- [ ] **第11页后**：收集学员的重复工作场景（5分钟）
  - 准备白板或在线文档记录
  - 选择2-3个典型场景简要分析
  
- [ ] **第25页后**：确认学员安装成功（2分钟）
  - "成功安装的举手"
  - 助教协助未完成的学员
  
- [ ] **第43页后**：讨论测试结果（3分钟）
  - "谁的Skill成功触发了？"
  - "遇到什么问题？"

### 应变准备
- [ ] **时间超时**：知道哪些内容可以快速跳过
- [ ] **学员提前完成**：准备进阶任务
- [ ] **学员跟不上**：准备简化版任务
- [ ] **技术故障**：立即切换到录屏演示

## 🆘 应急预案

### 技术故障
| 问题 | 预案 |
|-----|------|
| 投影仪故障 | 使用备用设备或学员自己的屏幕 |
| 网络断开 | 使用离线演示录屏 |
| Claude Code崩溃 | 切换到claude.ai网页版 |
| PPT文件损坏 | 使用PDF备份版本 |

### 时间问题
| 情况 | 预案 |
|-----|------|
| 严重超时 | 跳过第六部分，课后提供文档 |
| 严重提前 | 补充Q&A环节或进阶内容 |
| 某环节卡住 | 跳到下一环节，课后补充 |

### 学员问题
| 情况 | 预案 |
|-----|------|
| 无法安装软件 | 使用claude.ai网页版替代 |
| 大部分学员跟不上 | 降低难度，简化任务 |
| 提出超纲问题 | 记录下来，承诺课后解答 |
