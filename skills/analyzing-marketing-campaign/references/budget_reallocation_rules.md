## 规则 0：最低数据门槛

只有在分析周期内获得 **>=50 次转化 (conversions)** 的渠道，才具备调整预算的资格。

低于此门槛的渠道：直接归类为 **数据不足 (INSUFFICIENT_DATA) -> 维持 (MAINTAIN)**。

---

## 规则 1：渠道分类

将每个渠道归类为且仅归类为一个类别。请**按顺序**应用规则（一旦匹配成功即停止）：

### 暂停 (PAUSE) -> 减少 100%

任意条件触发：

* 用户指出该渠道已连续 3 周以上表现负向/亏损
* 用户指出在经过 3 周以上的优化努力后仍无改善

### 大幅削减 (DECREASE_HEAVY) -> 减少 45%

任意条件触发：

* ROAS < 目标值的 50% **且** 净利润 <= 0
* CPA > 上限值的 150% **且** 净利润 <= 0
* 三项全败：ROAS < 100% (未达标) **且** CPA > 100% (超标) **且** 净利润 <= 0
* 用户指出该渠道已连续 2 周表现负向/亏损

### 增加 (INCREASE) -> 使用用户指定的上限（默认 15%）

必须满足所有条件：

* ROAS >= 目标值的 115%
* CPA <= 上限值的 80%
* 净利润 > 0

### 小幅削减 (DECREASE_LIGHT) -> 减少 25%

任意条件触发（如果尚未被归类）：

* ROAS < 目标值的 80%
* CPA > 上限值的 120%

### 维持 (MAINTAIN) -> 变更 0%

不满足以上任何标准。

## 规则 2：计算预算变更

### 第 1 步：计算削减额 (全额应用)

```
大幅削减 (DECREASE_HEAVY): 削减额 = 当前花费 * 0.45
小幅削减 (DECREASE_LIGHT): 削减额 = 当前花费 * 0.25

```

### 第 2 步：计算释放的预算

```
释放预算 (freed_budget) = 所有削减额的总和

```

### 第 3 步：分配给“增加”类别的渠道

按净利润比例进行分配：

```
权重 (weight) = 该渠道净利润 / 所有“增加”类渠道的净利润总和
建议增加额 (proposed_increase) = 释放预算 * 权重

```

### 第 4 步：应用上限限制

**单渠道上限 (Per-channel cap)：** 如果用户提供了数值则使用，否则默认为 15%。

```
最大增加额 (max_increase) = 当前花费 * 增加上限比例(increase_cap)
最终增加额 (final_increase) = 最小值(建议增加额, 最大增加额)

```

**用户总调配限额 (User reallocation limit)**（如果指定）：

* 仅适用于 **增加 (increases)** 部分，不影响削减。
* 如果 `所有建议增加额的总和` > `用户限额`：则按比例缩放增加额。

```
缩放因子 (scale_factor) = 用户限额 / 所有建议增加额的总和
最终增加额 (final_increase) = 建议增加额 * 缩放因子

```

### 第 5 步：计算未分配结余

```
未分配金额 (unallocated) = 释放预算 - 所有最终增加额的总和

```

在报告中将其标记为“可用储备金 (available for reserve)”。

## 多周调整逻辑

如果用户提供了历史背景信息，请调整分类逻辑：

| 用户陈述 | 调整操作 |
| --- | --- |
| "某渠道已连续 2 周以上表现负向" | 升级为 **大幅削减 (DECREASE_HEAVY)** 或 **暂停 (PAUSE)** |
| "某渠道已连续 3 周以上表现负向" | **暂停 (PAUSE)** (预算设为 $0) |
| "我们上周刚调整了某渠道的预算" | 强制覆盖为 **维持 (MAINTAIN)** (给予 5-7 天稳定期) |
| "某渠道较上周有所改善" | 可将 **小幅削减 (DECREASE_LIGHT)** 升级为 **维持 (MAINTAIN)** |

如果没有历史背景，仅使用单周分类逻辑。

## 决策矩阵 (Decision Matrix)

| 条件 | 分类 | 变更幅度 |
| --- | --- | --- |
| 负向表现 3 周以上 (用户陈述) | 暂停 (PAUSE) | -100% |
| 无改善 3 周以上 (用户陈述) | 暂停 (PAUSE) | -100% |
| 负向表现 2 周 (用户陈述) | 大幅削减 (DECREASE_HEAVY) | -45% |
| ROAS < 50% 且 净利润 <= 0 | 大幅削减 (DECREASE_HEAVY) | -45% |
| CPA > 150% 且 净利润 <= 0 | 大幅削减 (DECREASE_HEAVY) | -45% |
| ROAS < 100% 且 CPA > 100% 且 净利润 <= 0 | 大幅削减 (DECREASE_HEAVY) | -45% |
| ROAS >= 115% 且 CPA <= 80% 且 净利润 > 0 | 增加 (INCREASE) | +15% (上限) |
| ROAS < 80% | 小幅削减 (DECREASE_LIGHT) | -25% |
| CPA > 120% | 小幅削减 (DECREASE_LIGHT) | -25% |
| 其他所有情况 | 维持 (MAINTAIN) | 0% |

## 要求的输出格式

### 1. 分类表 (Classification Table)

| 渠道 | ROAS | 目标达成率 (%) | CPA | 达到上限率 (%) | 净利润 | 分类结果 |

### 2. 计算步骤

展示：释放的预算、分配权重、建议增加额 vs. 上限后增加额、未分配的结余。

### 3. 最终调整表 (Final Reallocation Table)

| 渠道 | 当前预算 | 变更额 | 新预算 | 分类 |

如果有未分配金额 (> 0)，请包含“储备金 (Reserve)”一行。
